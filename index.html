About ACTIV8

Features

Fast Trials & Quick Learning

Compatible with lean development and MVP approaches. Parameters easy to change over-the-air to find the right setup for measurements.

Micro-service Architecture

Rearrange processing between gateway and cloud based on business case. Localise the data to save energy, optimize bandwidth, customized usability of data (service based data fetching)

End-to-end Intelligence

Data processing in sensor computers and gateways. Flexible cloud solution can store data. Fault tolerant two way communication from sensor to user

Low Total Cost of Ownership

Fast development iterations enable short time-to-production ; trials can be started in days or weeks. Based on open standards and selected open source components. Monitoring can be added easily by retrofitting.

Usage Tracking

Case Study

Customer wanted to know, how much different tools are used on a construction site. Sensors were installed to selected tools and a gateway sent the data to cloud. When tools were out of range, the sensors still collected the data and sent it, when gateway was available again.

Building Monitoring

Case Study

Customer wanted to demonstrate that IoT deployments can be done quickly. In less than two weeks from the first meeting, a remote building was equipped with temperature and humidity sensors, and data visualization viewed in the management team. Interesting information about building usage hours was detected from the humidity data.
